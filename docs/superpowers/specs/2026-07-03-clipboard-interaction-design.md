# 剪贴板式迁移交互设计（Clipboard Interaction Design）

日期：2026-07-03
状态：已与用户确认方向，待用户审阅本文档

## 背景与目标

现有 UI 用一个全局 Copy/Move 下拉框决定迁移模式，用户拖拽时无法直观知道
"这次是复制还是移动"，存在歧义。本设计用 Windows 资源管理器的剪贴板语义
（右键复制/剪切/粘贴 + Ctrl+C/X/V + 拖拽=移动）替代下拉框，让操作本身表达
意图。

**核心原则：Execute 之前一切都是模拟。** 树内的所有操作（复制、剪切、粘贴、
拖拽）只改变界面状态，体验上如同操作真实文件系统，但磁盘上一个字节都不会
改变。只有按下 `Execute` 并通过确认框后才真正落盘；落盘沿用现有安全模型
（所有涉及的安装根先全量备份，任一步失败全部回滚）。重新 `Scan` 即放弃全部
暂存。

已确认的三个决策：

1. 保持"暂存 + Execute"模型，粘贴不即时落盘。
2. 剪贴板操作只作用于**对话**（支持多选）；Code 分组仍用拖拽排序/换账户。
3. 反馈强度采用方案 B（剪切变淡 + 树内徽标 + 状态栏提示 + Execute 确认框）。

## 交互模型

| 操作 | 触发方式 | 效果 |
|------|----------|------|
| 复制 | 右键菜单 / `Ctrl+C` | 选中对话进入剪贴板（copy 模式），源条目不变 |
| 剪切 | 右键菜单 / `Ctrl+X` | 选中对话进入剪贴板（cut 模式），源条目变淡显示 |
| 粘贴 | 右键菜单 / `Ctrl+V` | 按剪贴板模式把对话暂存到目标位置 |
| 拖拽 | 鼠标拖放 | 等价"剪切 + 粘贴"（即移动），保留用于分组排序 |
| `Esc` | 键盘 | 清空剪贴板，被剪切条目恢复正常显示 |

### 粘贴目标语义

- 粘贴到 **Code 分组**：追加到该组末尾。
- 粘贴到 **账户**：进入该账户的 Ungrouped 组末尾。
- 粘贴到 **对话**：插入到该对话之后（同组）。

### 剪贴板规则

- 剪切后粘贴一次即清空剪贴板（Explorer 惯例）；复制可反复粘贴，每次产生一个
  新副本。
- 新的复制/剪切操作覆盖旧剪贴板内容。
- 剪切后粘贴回原位置 = 无操作（清空剪贴板，恢复显示）。
- 粘贴目标非法（空剪贴板、粘贴到被剪切项自身）时忽略操作并在状态栏说明。
- 复制粘贴到源所在组也允许（语义：给对话做一个副本）。
- **Copy/Move 下拉框删除**。迁移模式由操作本身决定，不再有全局模式。

## 落盘语义（Execute 时）

| 暂存操作 | 落盘行为 |
|----------|----------|
| 剪切粘贴 / 拖拽到**其他账户或其他安装根** | `migrate_sessions(MOVE)`：物理移动元数据文件 |
| 剪切粘贴 / 拖拽**同账户内** | 只更新分组布局（`claude_desktop_config.json`），不动文件（沿用现有逻辑） |
| 复制粘贴 | `migrate_sessions(COPY)`：创建副本，副本获得新 `sessionId`；原件及其分组完全不动 |

- 同一次 Execute 可**混合**移动与复制：`PlannedSessionMove` 增加 `mode` 字段
  （`MigrationMode`），`execute_plan` 的批次分组键加入 mode。
- 复制粘贴在树内产生"幽灵副本"条目（见下），幽灵副本不参与
  `staged_code_group_layout_by_root` 的 visible/assignments（沿用现有
  `exclude_session_ids` 机制）；Execute 后重新扫描，真实副本（新 id）出现，
  初始为未分组。

## 视觉反馈（方案 B）

1. **剪切即变淡**：被剪切对话的前景色变灰（粘贴或 Esc 后恢复）。
2. **树内徽标**：只有会产生**文件操作**的暂存条目在第二列显示状态——
   - 跨账户/跨安装根移动来的对话：`待移入`，蓝色；
   - 复制产生的幽灵副本：`⊕ 待复制`，绿色；
   - 同账户内的重分组（纯布局调整，不动文件）**不加徽标**，该类变更由
     状态栏与确认框中的"更新分组布局"体现。
3. **状态栏动词提示**：
   - 复制/剪切后："已复制/剪切 N 个对话——右键目标分组粘贴 (Ctrl+V)"；
   - 粘贴后："已暂存 N 个移动 + M 个复制。按 Execute 写入磁盘，重新 Scan 放弃。"
4. **Execute 确认框**："将移动 N 个对话、创建 M 个副本、更新分组布局。执行前
   会自动备份所有涉及的目录。继续？" 取消则不做任何事。
5. **Dry-run** 输出与确认框一致的摘要文本。

## 组件设计

### SessionTreeWidget（gui.py）

- 新增剪贴板状态：`clipboard_mode: MigrationMode | None` 与
  `clipboard_items: list[QTreeWidgetItem]`。
- `contextMenuEvent`：按右键目标类型动态构建菜单（对话→复制/剪切；
  分组/账户/对话→粘贴，剪贴板为空时禁用）。
- `keyPressEvent`：`Ctrl+C`/`Ctrl+X`/`Ctrl+V`/`Esc`。
- `paste_to(target_item)`：实现三种目标语义；剪切=移动现有条目，
  复制=创建幽灵副本条目。
- 幽灵副本条目：与源条目同样的 `ClaudeSession` 载荷，另加角色标记
  `UserRole+3 = "copy"`；移动条目标记 `UserRole+3 = "move"`。
  `_populate_trees`（重扫）时全部清除。
- 拖拽逻辑不变，仍走现有 `dropEvent`/`move_items_to_target`。

### MainWindow（gui.py）

- 删除 `mode_combo`。
- `planned_session_moves()`：跨账户/跨根的条目按 `UserRole+3` 得出各自
  mode（无标记的拖拽移动=MOVE）；幽灵副本无论目标在哪都生成 COPY 计划
  （包括同账户内——复制永远要落盘创建文件）。
- `execute_plan()`：
  - 批次分组键加 `mode`；`excluded_session_ids` 覆盖所有幽灵副本；
  - 弹出确认框（摘要 + 备份说明），确认后执行；
  - 其余备份/回滚/布局写入逻辑不变。
- 状态栏文案按上节实现。

### 不变的部分

- migrator.py / backup.py / code_groups.py / paths.py 均无需修改
  （migrate_sessions 已支持按批次传 mode，COPY 已改写 sessionId）。

## 错误处理

- Execute 失败：现有多根回滚模型不变；状态栏报告回滚结果。
- 剪切的对话被再次复制/剪切：新剪贴板覆盖，旧剪切项恢复显示。
- 幽灵副本被再次剪切/复制：禁止（右键菜单中禁用复制/剪切，快捷键忽略），
  避免"复制一个尚未落盘的副本"这种嵌套暂存。
- 撤销单个幽灵副本：右键菜单提供"撤销此暂存副本"，从树中移除该条目
  （等价于从未粘贴过）。`Esc` 只清空剪贴板，不撤销已粘贴的暂存。

## 测试计划

- 剪贴板状态机：复制/剪切/粘贴/Esc/覆盖，剪切变淡与恢复。
- 三种粘贴目标语义（分组/账户/对话）。
- 剪切粘贴回原位 = 无操作。
- 幽灵副本：树内出现、不污染 staged layout、Execute 后落盘为新 id 文件、
  "撤销此暂存副本"。
- 混合模式 Execute：同一批含移动 + 复制，各自落盘正确。
- Execute 确认框：取消则磁盘无任何变化。
- 快捷键 smoke（QTest 键盘事件）。
- 现有测试适配：删除 mode_combo 相关引用。

## 不在范围内（YAGNI）

- Code 分组整组的剪贴板操作。
- 变更面板与单条撤销（方案 C）。
- LevelDB 布局写入（已知限制，见 README）。
- 会话删除、重命名等其他文件管理操作。
